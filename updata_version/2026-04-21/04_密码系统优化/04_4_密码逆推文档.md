# 密码逆推文档

## 1. 密码逆推概述

密码逆推是指通过加密后的密文，反向推导出原始密码的过程。本文档将详细介绍如何逆推本系统中加密的密码，包括 AES 加密和简单加密两种方式。

## 2. 密码系统分析

### 2.1 加密流程

1. **密钥处理**：
   - 过滤密钥中的数字，生成密钥数字列表
   - 使用 SHA-256 对密钥进行哈希，生成 32 字节的 AES 密钥

2. **AES 加密**：
   - 生成随机 IV（16字节）
   - 对明文进行 PKCS7 填充
   - 使用 AES-CBC 模式加密
   - 将 IV 和密文组合，Base64 编码

3. **简单加密**（备用）：
   - 对明文中的每个字符进行移位操作
   - 反转结果字符串

### 2.2 存储格式

- 密码以密文形式存储在 JSON 文件中
- 密文是 Base64 编码的字符串
- 系统使用相同的密钥进行加密和解密

## 3. AES 加密密码逆推

### 3.1 所需信息

- **加密后的密文**：从存储文件中获取
- **加密密钥**：从配置文件中获取（`load_path["key"]["diary"]`）
- **加密算法**：AES-256 CBC 模式

### 3.2 逆推步骤

1. **获取密文**：
   - 从 `data/diary.json` 文件中找到目标日记的 `password` 字段
   - 复制加密后的密文字符串

2. **获取密钥**：
   - 从 `config/file_path.json` 文件中找到 `key` -> `diary` 字段
   - 复制密钥字符串

3. **解密密文**：
   - 使用 `SimpleCryptor` 类的 `decrypt` 方法解密密文
   - 具体步骤：
     1. 创建 `SimpleCryptor` 实例，传入密钥
     2. 调用 `decrypt` 方法，传入密文
     3. 获取返回的明文密码

### 3.3 代码示例

```python
from compents.simple_cryptor import SimpleCryptor

# 从配置文件中获取密钥
key = "your_key_here"

# 从存储文件中获取密文
encrypted_password = "your_encrypted_password_here"

# 创建加密器实例
cryptor = SimpleCryptor(key=key)

# 解密密码
decrypted_password = cryptor.decrypt(encrypted_password)

print(f"原始密码：{decrypted_password}")
```

### 3.4 注意事项

- AES 加密使用了随机 IV，因此相同密码加密后得到的密文不同
- 但解密过程不受 IV 影响，只要使用相同的密钥，就能正确解密
- 如果解密失败，可能是密钥不正确或密文损坏

## 4. 简单加密密码逆推

### 4.1 所需信息

- **加密后的密文**：从存储文件中获取
- **加密密钥**：从配置文件中获取（`load_path["key"]["diary"]`）
- **字符集**：系统默认字符集

### 4.2 逆推步骤

1. **获取密文**：
   - 从 `data/diary.json` 文件中找到目标日记的 `password` 字段
   - 复制加密后的密文字符串

2. **获取密钥**：
   - 从 `config/file_path.json` 文件中找到 `key` -> `diary` 字段
   - 复制密钥字符串

3. **解密密文**：
   - 使用 `SimpleCryptor` 类的 `decrypt` 方法解密密文
   - 具体步骤：
     1. 创建 `SimpleCryptor` 实例，传入密钥
     2. 调用 `decrypt` 方法，传入密文
     3. 获取返回的明文密码

### 4.3 手动逆推方法

如果没有代码环境，也可以手动逆推简单加密的密码：

1. **反转密文字符串**：
   - 简单加密的第一步是反转字符串，所以逆推的第一步是再次反转

2. **提取密钥数字**：
   - 过滤密钥中的数字，生成密钥数字列表
   - 例如：密钥 "my_key_123" 会提取出 [1, 2, 3]

3. **逐字符移位**：
   - 对每个字符，查找其在字符集中的索引
   - 使用密钥数字进行反向移位（索引 - 密钥数字）
   - 对结果取模字符集长度（处理负数）
   - 找到新索引对应的字符

4. **字符集**：
   ```
   0123456789ABCDEFGHIJKLMNOPQRSTUVWXYZabcdefghijklmnopqrstuvwxyz!@#$%^&*()_+-=[]{}|;:,.<>?/~`
   ```

### 4.4 示例

**假设**：
- 密钥："123"
- 密文："321dcba"

**逆推过程**：
1. 反转密文："abc321"
2. 提取密钥数字：[1, 2, 3]
3. 逐字符处理：
   - 'a' -> 索引 36 -> 36 - 1 = 35 -> 'z'
   - 'b' -> 索引 37 -> 37 - 2 = 35 -> 'z'
   - 'c' -> 索引 38 -> 38 - 3 = 35 -> 'z'
   - '3' -> 索引 3 -> 3 - 1 = 2 -> '2'
   - '2' -> 索引 2 -> 2 - 2 = 0 -> '0'
   - '1' -> 索引 1 -> 1 - 3 = -2 -> 取模 94 得到 92 -> '`'
4. 最终明文："zzz20`"

## 5. 密码逆推工具

### 5.1 命令行工具

可以创建一个简单的命令行工具来逆推密码：

```python
#!/usr/bin/env python3
"""
密码逆推工具
使用方法：python decrypt_password.py <encrypted_password> <key>
"""

import sys
from compents.simple_cryptor import SimpleCryptor

def main():
    if len(sys.argv) != 3:
        print("使用方法：python decrypt_password.py <encrypted_password> <key>")
        return
    
    encrypted_password = sys.argv[1]
    key = sys.argv[2]
    
    cryptor = SimpleCryptor(key=key)
    decrypted_password = cryptor.decrypt(encrypted_password)
    
    print(f"加密密码：{encrypted_password}")
    print(f"解密密钥：{key}")
    print(f"原始密码：{decrypted_password}")

if __name__ == "__main__":
    main()
```

### 5.2 使用方法

1. 保存上述代码为 `decrypt_password.py`
2. 运行命令：
   ```
   python decrypt_password.py <encrypted_password> <key>
   ```
3. 查看输出的原始密码

## 6. 密码逆推的安全性

### 6.1 安全风险

- **密钥泄露**：如果密钥被泄露，所有加密的密码都可以被逆推
- **密文泄露**：如果密文被泄露，结合密钥可以逆推原始密码
- **暴力破解**：对于弱密码，可能受到暴力破解攻击

### 6.2 防御措施

- **保护密钥**：确保配置文件中的密钥不被泄露
- **使用强密码**：使用包含字母、数字和特殊符号的复杂密码
- **定期更换密钥**：定期更新加密密钥
- **密码哈希**：考虑使用哈希存储密码，而不是加密存储

## 7. 常见问题

### 7.1 解密失败

**原因**：
- 密钥不正确
- 密文损坏
- 使用了错误的解密方法

**解决方法**：
- 检查密钥是否正确
- 确保密文完整无损坏
- 尝试使用简单加密进行解密

### 7.2 密码长度

**问题**：解密后的密码长度与预期不符

**原因**：
- PKCS7 填充的影响
- 简单加密的字符映射

**解决方法**：
- 检查解密结果是否包含填充字符
- 确认字符集是否正确

### 7.3 特殊字符

**问题**：解密后的密码包含特殊字符

**原因**：
- 简单加密的字符映射
- 填充字符

**解决方法**：
- 检查解密结果是否为预期的密码
- 确认字符集是否正确

## 8. 总结

密码逆推是一个基于加密算法的反向过程，需要知道加密密钥和加密算法才能成功逆推原始密码。本系统使用了 AES-256 和简单加密相结合的方式，在保证安全性的同时，也提供了相对简单的逆推方法。

通过本文档介绍的方法，可以成功逆推本系统中加密的密码。然而，需要注意的是，密码逆推应该仅用于合法目的，如忘记密码后的自我恢复，而不是用于非法获取他人密码。

为了确保密码安全，建议用户使用强密码，并妥善保管密钥和密码，避免密码泄露的风险。
