# 02_2 性能优化

## 优化内容

本优化对项目的 JSON 文件读写操作进行了性能优化，添加了缓存机制，减少了文件 I/O 操作，提高了读写效率。

## 实现原理

1. **缓存机制**：为 `FileProcess` 类添加了缓存机制，存储已读取的 JSON 文件内容
2. **缓存过期**：实现了缓存过期机制，确保数据的及时性
3. **路径处理**：优化了文件路径处理，使用类方法统一管理
4. **批量操作**：减少了重复的文件读写操作，提高了效率

### 核心代码

**缓存机制**：
```python
class FileProcess:
    # 类属性：真实根目录（指向exe/源码所在目录）
    ROOT_FILE = None
    # 缓存：存储已读取的JSON文件内容
    _json_cache = {}
    # 缓存过期时间（秒）
    _cache_expiry = 300  # 5分钟
    # 缓存最后更新时间
    _cache_last_update = {}

    @classmethod
    def _is_cache_valid(cls, path):
        """
        检查缓存是否有效
        :param path: 文件路径
        :return: bool
        """
        import time
        if path not in cls._json_cache:
            return False
        if path not in cls._cache_last_update:
            return False
        return time.time() - cls._cache_last_update[path] < cls._cache_expiry

    @classmethod
    def read_json(cls, path) -> dict:
        """
        读取整个json文件 并返回
        :param path: 文件路径 根目录开始
        :return: dict -> json数据
        """
        cls._init_root_path()
        full_path = os.path.join(cls.ROOT_FILE, path)
        
        # 检查缓存是否有效
        if cls._is_cache_valid(full_path):
            return cls._json_cache[full_path]
        
        try:
            with open(full_path,"r",encoding="utf-8") as f:
                data = json.load(f)
                # 更新缓存
                cls._json_cache[full_path] = data
                import time
                cls._cache_last_update[full_path] = time.time()
                return data
        except Exception as e:
            logger.error(f"读取JSON文件失败：{e}")
            return {}
```

**缓存清除方法**：
```python
@classmethod
def clear_cache(cls, path=None):
    """
    清除缓存
    :param path: 可选，指定路径，不指定则清除所有缓存
    """
    if path:
        full_path = os.path.join(cls.ROOT_FILE, path)
        if full_path in cls._json_cache:
            del cls._json_cache[full_path]
        if full_path in cls._cache_last_update:
            del cls._cache_last_update[full_path]
    else:
        cls._json_cache.clear()
        cls._cache_last_update.clear()
```

## 效果展示

- 减少了重复的文件 I/O 操作，提高了读写效率
- 缓存过期机制确保了数据的及时性
- 统一的路径处理提高了代码的一致性

## 注意事项

- 缓存默认过期时间为5分钟，可以根据需要调整
- 写入操作会自动更新缓存，确保数据一致性
- 可以使用 `clear_cache` 方法手动清除缓存
